# Phishing Kit Infrastructure Patterns

<sub>[← Back to IoCs](../) · [← Back to Threat-Hunting](../../)</sub>

Signatures, URL patterns, and infrastructure fingerprints for active phishing kits — especially adversary-in-the-middle (AitM) tooling that defeats MFA. Pattern-based rather than atomic: the kits themselves are persistent, the domains hosting them rotate within 24 hours.

## Usage

| Platform | How to apply |
|---|---|
| **Microsoft Sentinel** | KQL regex on `SigninLogs.OriginalRequestUri` and `AuditLogs` for these patterns; alert on sign-in from unverified publisher OAuth app with matching URL |
| **Defender for Cloud Apps (MDA)** | Custom policy: New / uncommon reply URL + unverified publisher |
| **Proxy / SWG** | Regex deny for the URL-path signatures below |
| **Email gateway** | URL rewriting + sandbox detonation for domains matching these patterns |

## Indicators

| Pattern | Kit / Campaign | First Published | Confidence | Source | Notes |
|---|---|---|---|---|---|

| Presence of a decodable base64-encoded GET parameter in a suspicious login URL | Evilginx (custom parameter encoding) | 2024 | High | [Breakdev — Evilginx custom parameters](https://breakdev.org/evilginx-pro-4-2/) | If any query-string param decodes cleanly as base64, the URL was generated by Evilginx (public encoder source) |
| URL expires within 24 hours of sending + unique per recipient | Evilginx3 / EvilPanel AitM | 2025-04 | High | [Abnormal AI — EvilPanel](https://abnormal.ai/blog/evilpanel-automated-aitm-phishing) · [CyberPress — Evilginx attack techniques](https://cyberpress.org/evilginx-attack-techniques/) | 18+ higher-ed institutions targeted with Evilginx3 since April 2025. ## Notes

- **Evilginx kits request Let's Encrypt certs** — so certificate issuance alerts on lookalike domains get you the earliest detection window.
- AitM short-circuits MFA by stealing the **session cookie**, not the password. Token-protection (device-bound tokens) and phishing-resistant MFA (FIDO2) are the real mitigations; detection alone is a losing race.
- The "decode every GET parameter" Evilginx heuristic has a meaningful false-positive rate — treat matches as "investigate," not "block."
